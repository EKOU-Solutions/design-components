# DataTable

An Excel-like data grid built for accounting and tabular workflows. Supports keyboard navigation (ARIA grid pattern), inline cell editing, auto-computed totals, row creation, deletion, and disabled rows.

---

## Architecture

```
DataTable (DataTable.tsx)
├── Toolbar            — "+ Add row" button (shown when onAddRow is provided)
├── scroll wrapper     — overflow: auto, bounded by height or fill-parent
│   └── <table role="grid">
│       ├── <thead>   — ColumnHeader × N + optional Actions th
│       └── <tbody>   — DataRow × N
│           └── Cell / CellEditor / SelectEditor per cell
└── totals wrapper     — width: 100%, always full container width
    └── <table>        — table-layout: fixed, TotalsRow
```

**Key design decisions:**

- **Data-down, events-up.** DataTable owns no data. All mutation is signalled via callbacks (`onCellChange`, `onAddRow`, `onDeleteRow`). The parent controls state.
- **Two-table layout for totals.** The totals footer lives in a separate `<table>` outside the scroll container so it is always visible and always fills `100%` of the container width regardless of the main table's content width.
- **ARIA grid pattern.** The `<table>` has `role="grid"` and `aria-activedescendant`. Cells are identified by `id="cell-{rowIndex}-{colIndex}"`. Keyboard navigation follows the [ARIA grid spec](https://www.w3.org/WAI/ARIA/apg/patterns/grid/).
- **CSS custom properties for theming.** All visual tokens (`--table-bg`, `--table-border`, `--table-header-bg`, etc.) are defined in `tokens.css` and applied via CSS modules.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `columns` | `ColumnDef[]` | — | Column definitions (see below) |
| `rows` | `RowData[]` | — | Row data. Each row must have a unique `id: string` |
| `height` | `number \| "fill"` | — | Fixed max-height in px, or `"fill"` to stretch to parent |
| `className` | `string` | — | Additional class on root element |
| `initialFocusedCell` | `{ rowIndex, colIndex }` | — | Pre-focus a cell without user interaction |
| `onCellChange` | `(rowId, colKey, value) => void` | — | Called when a cell value is committed |
| `onAddRow` | `() => void` | — | Shows the + toolbar button; called on click |
| `deletable` | `boolean` | `false` | Adds a delete column with a trash icon per row |
| `onDeleteRow` | `(rowId) => void` | — | Called when the trash icon is clicked |
| `getRowDisabled` | `(row: RowData) => boolean` | — | Return `true` to make a row read-only and skip it during keyboard navigation |

### ColumnDef

```ts
interface ColumnDef {
  key: string;          // unique column key, matches RowData property names
  label: string;        // column header text
  width?: number;       // px width (default: 100)
  align?: 'left' | 'right' | 'center';
  type?: 'string' | 'number' | 'select';  // controls editor and totals sum
  options?: SelectOption[];  // required when type='select'
  editable?: boolean;   // false disables edit mode for this column (default: true)
  placeholder?: string; // shown in text editor when value is empty
  showTotal?: boolean;  // include column in the totals footer row
  totalLabel?: string;  // static label in totals cell (e.g. "TOTAL:"); if omitted, sums numbers
}
```

### RowData

```ts
interface RowData {
  id: string;           // must be unique — used as React key and in callbacks
  [key: string]: unknown;
}
```

## Keyboard Navigation

| Key | Action |
|-----|--------|
| Arrow keys | Move focused cell |
| `Home` / `End` | Jump to first / last column in current row |
| `Ctrl+Home` / `Ctrl+End` | Jump to first / last cell in the grid |
| `Enter` / `F2` | Enter edit mode on focused cell |
| `Escape` | Cancel edit, return to navigation mode |
| `Tab` (in edit mode) | Commit value and move to next editable cell |
| `Enter` (in delete column) | Trigger `onDeleteRow` for focused row |

Disabled rows are skipped during `ArrowUp` / `ArrowDown` navigation and cannot enter edit mode.

---

## Totals Footer

Set `showTotal: true` on any column to display the totals row.

- If `type: "number"` — the column value is summed. Commas are stripped before parsing (e.g. `"1,234.56"` → `1234.56`).
- If `totalLabel` is set — that static string is shown (useful for labelling the row, e.g. `"TOTAL:"`).
- Otherwise — the cell is empty.

The totals table is a separate element outside the scroll area. It always fills `100%` of the DataTable container width.

---

## Cell Editors

**Text / Number (`type: "string" | "number"`)** — renders an `<input>`. Commits on `Enter` or blur. Cancels on `Escape`.

**Select (`type: "select"`)** — renders a custom ARIA listbox (`role="listbox"`, `role="option"`). `ArrowDown` / `ArrowUp` navigate options, `Enter` commits, `Escape` cancels. Arrow keys do not propagate to the table's scroll container.

## Theming

Override any CSS custom property on a parent element to theme the table:

```css
.my-section {
  --table-bg: #1e1e1e;
  --table-border: #444;
  --table-header-bg: #2d2d2d;
  --table-header-color: #f0f0f0;
  --table-row-hover-bg: #2a2a2a;
  --table-selected-bg: #2c3e50;
  --table-totals-bg: #2d2d2d;
  --table-font-size: 12px;
}
```

All tokens are declared in `tokens.css` on `:root`.
